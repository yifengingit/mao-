# 可视化: 设计思路与当前状态

## 核心设计决策

### "把时间轴移到节点上，而不是把节点移到时间轴上"

传统做法（如 KronoGraph）是把节点重新排列到时间轴上，但这会破坏力导向图的社群结构。
我们的做法是: 保持空间布局不变，用时间滑块做渐进式过滤。拖动 1925->1957，节点和边随时间"涌现"。

### 三级密度

1068 个节点在力导向图里是一坨毛线团。解决方案:
- **精要** (默认): degree >= 5, 只显示 99 个核心节点。可读、美观。
- **扩展**: degree >= 3, 194 个节点。中等密度。
- **全量**: 全部 1,068 个节点。学术用途。

### 传记模式 (Biography Machine)

选择任意节点 -> 图谱变暗，只显示该节点的 ego-network -> 按年份播放，看该人物的关系网络随时间积累。

每个节点都是自己的纪录片。

### 来源段落 (Article Drawer)

点击任何边，右侧面板展示 evidence_items: 原文引用 + 出处文章 + 日期。
这是把"好看的可视化"变成"可信的研究工具"的关键功能。

## v1 -> v2 变化原因

用户反馈 v1 "感觉一般，没有什么吸引力和可读性"。

v1 问题:
1. 1068 个节点全部显示 = 毛线团，看不出任何信息
2. 白色/浅色主题，不够专业
3. 没有 zoom-to-fit，节点散布在巨大的画布上
4. 力导向参数不适合 1000+ 节点

v2 改进:
1. 默认 99 节点 (精要模式)
2. GitHub 暗色主题
3. 自动 zoom-to-fit (simulation.on("end"))
4. 自适应力参数: 节点少时 charge=-200 (间距大), 多时 charge=-50

## 待测试 / 待修复

1. **zoom-to-fit 居中**: 截图显示图谱偏画布下方，可能需要调整 forceCenter 强度或排除 timeline 高度
2. **密度切换重建**: 每次切换密度会完全销毁并重建 simulation，性能可以优化为 toggle visibility
3. **传记模式播放**: 需要验证 setInterval 是否正确推进年份并更新 ego-network
4. **搜索**: 当前只做了 display:none 过滤，没有更新 simulation
5. **移动端**: 右面板需要改成 bottom sheet
6. **label 重叠**: 99 节点时所有标签都显示 (showAllLabels = visNodes.length <= 120)，可能部分重叠
