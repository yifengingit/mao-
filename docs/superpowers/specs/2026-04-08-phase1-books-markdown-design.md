# Phase 1 Design — 书籍素材入库、按卷拆分、PDF→Markdown、清洗验证

日期：2026-04-08
项目：`260407mao`
范围：只覆盖 Phase 1，不进入 LLM 抽取、知识图谱、网站产品化。

## 1. 目标

Phase 1 的目标不是做完整知识产品，而是打稳文献处理地基。

成功标准：
1. 大多数书籍 PDF 能稳定转成可读 Markdown。
2. 同时沉淀出一套可复用的清洗流程与质检流程。

明确不包含：
- 人物/事件/组织档案生成
- 概念卡片生成
- 知识图谱构建
- 网站展示与售卖

## 2. 已确认约束

- 第一阶段素材以书籍为主。
- 用户手动搜集素材。
- 入库策略采用最低摩擦方案：先将 PDF 直接放入 `data/raw/books/`，元数据后补。
- 当前首批样本只有一个：
  - `毛泽东选集(毛选1-7原版五卷+静火+赤旗+草堂) (毛泽东) (z-library.sk, 1lib.sk, z-lib.sk)`
- 该样本不直接整本处理，而是先按卷拆分。
- 最小处理单元是按卷，而不是按篇。

## 3. 推荐方案与取舍

### 方案 A：整本直转
优点：步骤最少。
缺点：超大 PDF 更容易失败；问题定位困难；清洗和质检成本高；后续无法稳定复用。

结论：不采用。

### 方案 B：按卷拆分后转
优点：
- 更稳定
- 更容易定位错误
- 更适合人工清洗与质检
- 后续也更适合卷级知识整理

缺点：
- 多了一步拆分与页码映射维护

结论：采用本方案。

### 方案 C：按篇拆分后转
优点：粒度最细。
缺点：前期整理成本过高，不适合 Phase 1。

结论：暂不采用。

## 4. Phase 1 正式工作流

### Step 1. 原始素材入库
- 将原始 PDF 放入 `data/raw/books/`
- 不修改原文件，不覆盖原文件
- 当前阶段允许先不补完整元数据
- 至少要能识别“这是哪一份总集文件”

### Step 2. 卷级拆分
将总集 PDF 拆成多个卷级 PDF。

拆分要求：
- 每卷一个 PDF
- 卷文件命名统一
- 记录每卷在原总 PDF 中的页码范围
- 保留卷级文件与原总 PDF 的回溯关系

本步骤是后续所有处理的正式输入边界。

### Step 3. 卷级转换
对每卷 PDF 执行 Markdown 转换。

默认策略：
- `pdf -> marker`
- 只在需要对照或排查时再用 `markitdown`

每卷至少保留：
- 原始 Markdown 输出
- 转换日志
- 失败原因或异常记录

### Step 4. 卷级清洗
在原始 Markdown 基础上做清洗，输出到 `data/clean/`。

优先清洗项：
- 页眉页脚
- 页码残留
- 断行
- OCR 噪音
- 目录误识别
- 脚注错位
- 表格或图片占位异常

清洗原则：
- 不改动原义
- 不覆盖原始 Markdown
- 清洗规则应可重复应用

### Step 5. 卷级质检
对每卷建立固定质检项：
- 标题是否完整
- 段落是否连贯
- 是否存在明显漏字或乱码
- 页码映射是否可回溯
- OCR 错误是否显著影响阅读
- 多栏/脚注/表格是否严重错乱

问题应打标签，便于后续总结失败模式。

### Step 6. 规则固化
当 1 卷跑通后，总结并固化：
- 卷级命名规范
- 拆分规则
- 清洗规则
- 质检清单
- 常见失败标签
- 哪些情况必须人工复核

然后再扩展到剩余卷。

## 5. 数据与目录边界

当前 Phase 1 相关目录：

- `data/raw/books/`：原始总 PDF 与卷级 PDF
- `data/staging/`：拆分中间文件、日志、临时产物
- `data/markdown/`：工具原始 Markdown 输出
- `data/clean/`：清洗后的 Markdown
- `data/metadata/`：卷级页码映射、最小元数据

建议补充的卷级信息：
- 卷编号
- 卷标题
- 原总 PDF 页码范围
- 是否扫描件
- 是否需要 OCR
- 转换状态
- 质检状态

## 6. 当前最小可用闭环

不追求一次性处理全部卷。

先做：
1. 识别总 PDF 的卷边界
2. 拆出 1 卷 PDF
3. 用 `marker` 转换这一卷
4. 做一轮人工清洗
5. 按质检表检查
6. 记录失败模式

当这一卷闭环通过后，再复制到其他卷。

## 7. 现在应该先做什么

1. 补一份卷级拆分规范
2. 先拆出第 1 卷
3. 验证第 1 卷的 `marker` 转换质量
4. 写出第一版清洗规则与问题标签

## 8. 暂时不要做什么

- 不做知识图谱 schema
- 不做网站信息架构
- 不做人物/事件/组织档案模板
- 不追求全量自动化批处理
- 不直接整本总 PDF 转换

## 9. 设计结论

Phase 1 应采用“单一重样本打通法”：

> 从这份总集 PDF 出发，按卷拆分，先跑通 1 卷完整闭环，再逐步扩展到其余卷。

这比一开始追求大批量处理、知识抽取、图谱建设更稳，也更符合当前阶段目标。
