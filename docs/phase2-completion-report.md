## 数据质量评估

### 优点
1. **高覆盖率**：73.9% 的文章成功处理
2. **高去重率**：人物实体去重率 67%，有效减少冗余

5. **关系丰富**：抽取了 1,176 个唯一关系，覆盖多种关系类型

### 局限性
1. **失败文章**：110 篇文章未成功处理（主要是 volume-06）
2. **API 稳定性**：DeepSeek API 返回的 JSON 格式不稳定，导致部分失败
3. **实体类型单一**：目前只完成了人物实体，地点、组织等类型因 API 失败率高而暂缓
4. **关系类型有限**：主要是人物-人物和人物-组织关系，缺少人物-事件、人物-地点等

## 技术总结

### 成功经验
1. **分阶段处理**：文章切分 -> 实体抽取 -> 去重 -> 关系抽取，流程清晰
2. **去重策略**：精确名称匹配 + 别名聚合，效果良好
3. **重试机制**：API 调用失败时自动重试，提高成功率
4. **数据验证**：每个阶段都有验证脚本，确保数据质量

### 遇到的问题
1. **API 不稳定**：DeepSeek API 返回的 JSON 格式不稳定
2. **内容过长**：部分文章内容过长，导致 API 超时
3. **编码问题**：Windows 终端编码导致日志显示乱码

### 改进方向
1. **JSON 修复机制**：实现自动修复常见 JSON 格式错误
2. **内容分段**：对过长文章进行分段处理
3. **错误日志**：记录原始 API 响应，便于调试
